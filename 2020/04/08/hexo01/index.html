GitHubPages+Hexo安装使用

2020-04-08
字数统计: 692   |   阅读时长≈ 2 分钟

心血来潮打算在Github搭建一个个人博客，把一些小项目记录下来，稍微研究之后，选中了Hexo（因为简单）。记录了一下在Win10环境安装Hexo并推送博客到Github的过程。关于Hexo就不过多介绍了，可以到官方网站查看。

1.安装依赖项

Git

下载对应版本安装即可，在cmd中执行git命令确定是否可用。
https://git-scm.com/download/win

Node.js

下载最新安装程序，在安装时不要取消ADD TO PATH选项（设置保持默认即可）。
https://nodejs.org/en/download/

2.安装Hexo

切换npm源

使用默认npm源安装基本没有速度，所以首先把npm源切换为国内淘宝镜像站。

4.推送到Github

建立仓库

新建公共仓库，名称一般命名为username.github.io，设置为空仓库即可。
在该仓库->Setting->GitHub Pages->Source中确认Your GitHub Pages site is currently being built from the master branch.
此时已经可以通过https://username.github.io访问该仓库源文件，接下来将Hexo生成的网站推送至仓库。

配置网站

重新打开_config.yml文件，主要修改deploy部分，为其添加一种发布方式，注意缩进。repo内容为该项目的git地址。当然，为了能顺利将网站push到github仓库，需要你已经配置过ssh密钥（即通过命令行的git有权限向自己的git仓库进行push），该部分可以参考Git与服务器配置。
